Extraction and transformation process

This functions take cares of the data transformation and data extraction activity. It preps the data and makes it ready for writing to big query.

src.data_engineering.consolidated.data_extraction.repo_github(repo, owner, api, extract)

This function reads from github api, it iterates over the pages and can bring the information about pull requests, commits, gist and much more.

Parameters

  • repo – The repo we want to extract the information for e.g. spark , pandas

  • owner – Owner of the repo e.g apache, pydata.

  • api – Credentials to acess the github api

  • extract – Information we need to extract pulls, commits etc

Returns

Extracted information in a dictionary format

src.data_engineering.consolidated.data_extraction.create_commits_df(repo, owner, api, extract)

Once the data is extracted from the git, it gets normalised and gets converted as a dataframe.

Parameters

  • repo – Repo which data needs to be extracted

  • owner – Owner of the repo e.g. apache , pydata

  • api – Your github credentials

  • function – Function to extract the pull, commits etc

  • extract –

Returns

pandas dataframe which is further used to write to big query
